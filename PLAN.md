# 구현 계획 (PLAN)

## 주차별 마일스톤

### 1주차 — 부품 확보 및 RFID/상품 DB
- 부품 구매 (Arduino Uno, RC522, NTAG213 스티커, I2C LCD, 엔코더, 부저, IR 센서, 릴레이, 서보)
- RC522 + NTAG213 스티커 단독 동작 테스트 (UID 읽기 안정성 확인)
- 상품 DB 구조 설계 (EEPROM 또는 코드 내 테이블)
- RFID UID ↔ 상품명/가격 매핑 구현 및 검증

### 2주차 — 장바구니 / 입력 / 결제
- 장바구니 로직 구현 (추가, 제거, 수량 변경, 합계 계산)
- 로터리 엔코더로 수량 조절 (회전 +/−, 버튼 확정)
- I2C LCD 연동 (스캔 화면, 장바구니 화면, 합계 화면)
- 결제 처리 (`isPaid` 플래그 전이, 결제 완료 LCD/부저 피드백)

### 3주차 — 게이트 통합
- IR 센서 게이트 통과 감지 (임계값 캘리브레이션)
- 릴레이 + 서보모터 게이트 개폐 제어 (0°/90°)
- `isPaid` 검사 → 미결제 시 부저 경고, 결제 시 게이트 오픈
- 6단계 시나리오 전체 통합 테스트

### 4주차 — 오류 수정 및 시연 준비
- 전체 회귀 테스트 (정상 결제 / 미결제 통과 / 취소 / 수량 변경)
- 엣지 케이스 수정 (중복 태그, EEPROM 누락 UID, 센서 오작동)
- 데모용 상품 5종 NTAG213 스티커 최종 부착
- 시연 시나리오 리허설 및 발표 자료 정리

---

## 변경 로그
계획이 변경되거나 구현이 막혔을 때 여기 기록한다. (밀린 일기 형식, 시간은 적지 않음)

### 초기 계획 수립
- 4주 계획 초안 작성. 1주차=센서/입력 검증, 2주차=액추에이터/게이트, 3주차=통합/상태머신, 4주차=테스트/발표.

### 주차 구성 재편
- 지도교수 안내에 맞춰 주차별 작업 단위를 "기능 흐름" 중심으로 재편.
- 1주차를 부품 구매와 RFID↔상품 DB 매핑으로 앞당기고, 액추에이터/게이트 작업은 3주차로 이동.
- 2주차는 장바구니·엔코더·LCD·결제까지 묶어 화면/입력 흐름을 한 번에 완성하도록 변경.
- 3주차는 IR + 릴레이 + 서보 게이트 통합 및 전체 시나리오 통합 테스트로 집중.
- 4주차는 오류 수정과 최종 시연 준비로 단순화 (발표 PPT 생성은 막바지에 결정).
